# рҹ“Ӣ Plan de Desarrollo: Cantoral PWA (Vue 3)

## рҹҺӯ Contexto y Rol

ActuarГЎs simultГЎneamente como:

- **Arquitecto de Software**: diseГұarГЎs la estructura, patrones y escalabilidad del proyecto.
- **Desarrollador Senior Vue/Frontend**: escribirГЎs cГіdigo limpio, modular y mantenible.
- **MГәsico Experto**: validarГЎs la lГіgica de acordes, transposiciГіn (latino/inglГ©s) y formato de cantorales.

### Principios rectores (NO NEGOCIABLES)

1. **CГіdigo limpio** siempre: nombres expresivos, funciones pequeГұas, SRP (Single Responsibility Principle).
2. **Patrones de diseГұo** donde aporten valor: Composable Pattern (Vue 3), Strategy (parsers/transposiciГіn), Observer (stores reactivos), Factory (creaciГіn de cantos), Repository (acceso a datos).
3. **Legibilidad > cleverness**. Comentarios SOLO cuando el "por quГ©" no sea obvio.
4. **Componentes atГіmicos y reutilizables** (Atomic Design: atoms вҶ’ molecules вҶ’ organisms).
5. **HTML y CSS nativos**, sin frameworks CSS. Variables CSS, `clamp()`, container queries, `@layer`.
6. **Mobile-first absoluto**. DiseГұo pensado para navegador mГіvil e instalaciГіn como PWA.
7. **Sin TypeScript**. JavaScript moderno (ES2022+) con JSDoc en funciones crГӯticas.

### Formato laCuerda (soporte completo del formato REAL):

El parser debe reconocer el formato real descargado desde `acordes.lacuerda.net/TXT/...`. Ejemplo canГіnico en `EJEMPLO_LACUERDA.TXT`:

```
=====================================================================
|             TABLATURAS Y ACORDES DE MГҡSICA EN ESPAГ‘OL             |
+-------------------------- lacuerda.net ---------------------------+
| ARTISTA: Gianluca Grignani                                        |
| CANCION: MI HISTORIA ENTRE TUS DEDOS                              |
| AUTOR:   Gianluca Grignani                                        |
| ALBUM:   Destinazione Paradiso                                    |
+------------------------------------------------------ACO-gian0042-+
| TRANS:   Marco                                                    |
=====================================================================

INTRO:    Dm7  -  G  -  Cmaj7 - Fmaj7

 Dm7          G            Cmaj7                      Fmaj7
Yo pienso que, no son tan inГәtiles las noches que te di
...

CORO
               C
      Hay una cosa que yo no te he dicho aГәn
...
```

### Reglas del `FormatDetector`:

1. Si el archivo empieza con `---` (frontmatter YAML), usa `NativeParser`.
2. Si el archivo empieza con `=====` o contiene `| ARTISTA:` / `| CANCION:`, usa `LaCuerdaParser`.
3. Fallback: si no hay header estructurado pero hay acordes sobre letra, tambiГ©n se procesa como laCuerda heurГӯstico.

### Reglas del `LaCuerdaParser`:

- **Bloque header** (entre lГӯneas `====`): extraer `ARTISTA` вҶ’ `artist`, `CANCION` вҶ’ `title` (normalizado a Title Case), `AUTOR` вҶ’ `author`, `ALBUM` вҶ’ `album`, `TRANS` (transcriptor) вҶ’ metadata ignorable o `transcriber`.
- **Bloque footer**: descartar disclaimer legal de lacuerda.net (todo tras la segunda lГӯnea de `====` al final).
- **Marcadores de secciГіn sin corchetes**: lГӯneas que contienen SOLO una palabra en mayГәsculas (`INTRO`, `CORO`, `PUENTE`, `SOLO`, `ESTROFA`, `VERSO`) se tratan como encabezado de secciГіn. TambiГ©n variantes con nГәmero (`CORO 2`, `VERSO 1`).
- **`INTRO:` inline**: si la lГӯnea empieza con `INTRO:` seguido de acordes separados por `-`, extraer la secuencia de acordes como una secciГіn `intro`.
- **LГӯneas de acordes vs letra**: heurГӯstica вҖ” una lГӯnea es de acordes si вүҘ70% de sus tokens no-vacГӯos matchean el regex `ENGLISH_CHORD` o `LATIN_CHORD`. Pares consecutivos `(lГӯnea_acordes, lГӯnea_letra)` se agrupan preservando alineaciГіn por columnas.
- **Tono (opcional)**: lГӯneas tipo `Tono: X`, `Tempo: X`, `Capo: X` si aparecen вҶ’ metadata. El formato lacuerda.net real no siempre las incluye (la tonalidad se infiere del primer acorde o se deja `null`).

Ambos parsers devuelven el **mismo objeto normalizado**:

```js
{
  id: 'como-el-ciervo',
  metadata: { title, artist, author, album, key, tempo, tags, categories },
  sections: [
    { type: 'intro', lines: [...] },
    { type: 'verse', label: 'Verso 1', lines: [...] },
    { type: 'chorus', label: 'Coro', lines: [...] }
  ],
  sourceFormat: 'native' | 'lacuerda'
}
```

## рҹҺө Motor de Acordes (Chord Engine)

### DetecciГіn (`ChordDetector.js`)

```js
// InglГ©s: C, C#, Db, Cm, Cmaj7, C7, Csus4, C/G, etc.
const ENGLISH_CHORD =
  /^[A-G](#|b)?(m|maj|min|dim|aug|sus)?\d?(\/[A-G](#|b)?)?$/;

// Latino: Do, Re#, Mim, Lam7, Do/Sol, etc.
const LATIN_CHORD =
  /^(Do|Re|Mi|Fa|Sol|La|Si)(#|b)?(m|maj|min|dim|aug|sus)?\d?(\/(Do|Re|Mi|Fa|Sol|La|Si)(#|b)?)?$/;
```

HeurГӯstica: una lГӯnea es "de acordes" si **вүҘ70% de sus tokens no vacГӯos** matchean el regex.

### TransposiciГіn (`ChordTransposer.js`)

- Array circular de 12 semitonos.
- `transpose(chord, semitones)` preserva calidad (`m`, `maj7`, `sus4`), bajo invertido (`/G`) y notaciГіn original.
- Respeta preferencia de sostenidos vs bemoles segГәn la tonalidad destino.

### ConversiГіn de notaciГіn (`NotationConverter.js`)

Mapa bidireccional: `C вҶ” Do`, `D вҶ” Re`, etc.

## рҹ“Ҙ GestiГіn de Cantos Offline

### Flujo al instalar la PWA:

1. **Primera carga (online)**: la app descarga `public/songs/index.json`.
2. **Al instalar la PWA** (evento `appinstalled`): se descargan automГЎticamente **TODOS los cantos** del manifest y se guardan en IndexedDB.
3. **Post-instalaciГіn**: el usuario puede abrir `OfflineManagerView` y ver:
   - Lista completa de cantos disponibles (del manifest remoto).
   - Estado de cada uno: вң… Descargado / в¬ҮпёҸ No descargado.
   - Toggle individual por canto o "Descargar todos" / "Eliminar todos".
   - Indicador de espacio usado.

### Componentes clave:

**`useOfflineManager.js`**:

```js
{
  (availableSongs, // Del manifest remoto
    downloadedSongs, // De IndexedDB
    isDownloaded(id),
    downloadSong(id),
    removeSong(id),
    downloadAll(),
    syncManifest()); // Refresca el manifest
}
```

**`offline.store.js`** (Pinia):

- Estado: `{ downloadedIds: Set, downloading: Set, lastSync: Date }`.
- Acciones disparan eventos que `OfflineManagerView` consume.

**Estrategia con Workbox**:

- `public/songs/index.json` вҶ’ `StaleWhileRevalidate`.
- Archivos `.song` вҶ’ **NO precacheados**; se descargan bajo demanda y guardan en IndexedDB (no en Cache API, para poder consultarlos como datos estructurados).

### UI de `OfflineManagerView.vue`:

- Header: "Cantos disponibles offline".
- Barra de progreso global al descargar.
- Lista virtualizada con checkbox, tГӯtulo, artista, tamaГұo.
- Botones: `Descargar todo`, `Eliminar todo`, `Refrescar manifest`.

## рҹ§Ә Testing con Playwright MCP

**вҡ пёҸ InstrucciГіn CRГҚTICA para la IA ejecutora**: usa el **MCP de Playwright** para **ejecutar y validar** las pruebas interactivamente, no solo para escribir los archivos de test. El flujo debe ser:

1. Escribir el test en `tests/e2e/*.spec.js`.
2. **Invocar el MCP de Playwright** para:
   - Abrir el navegador.
   - Navegar a la URL local (`http://localhost:5173`).
   - Ejecutar los pasos del test (click, fill, expect).
   - Tomar screenshots ante fallos.
   - Verificar que la PWA funciona offline (desconectar red y recargar).
3. Reportar resultados en lenguaje natural, no solo logs.

### Casos E2E mГӯnimos:

1. BГәsqueda de canto por tГӯtulo devuelve resultados.
2. Apertura de canto muestra letra + acordes correctamente alineados.
3. BotГіn de transposiciГіn Вұ1 cambia todos los acordes.
4. Cambio de notaciГіn latino вҶ” inglГ©s funciona sin perder transposiciГіn.
5. Filtrado por categorГӯa y subcategorГӯa.
6. App carga offline despuГ©s de primera visita.
7. Instalabilidad en mГіvil (verificar manifest).
8. **Parser laCuerda**: cargar un canto en formato laCuerda y verificar renderizado idГ©ntico al nativo.
9. **Offline manager**: descargar un canto, desconectar red, abrirlo.
10. **Al instalar**: verificar que todos los cantos del manifest quedan en IndexedDB.

### Tests unitarios (Vitest):

- `ChordDetector`: casos lГӯmite.
- `ChordTransposer`: 12 tonalidades.
- `NotationConverter`: ida y vuelta idempotente.
- `NativeParser` y `LaCuerdaParser`: archivos reales de cada formato.

Una feature estГЎ completa cuando:

1. Tiene tests unitarios (si es lГіgica) o story de Storybook (si es UI).
2. Pasa `oxlint` sin warnings.
3. Pasa `prettier --check`.
4. Funciona en mГіvil (viewport 360px mГӯnimo).
5. Es accesible por teclado.
6. Tiene manejo de estados (empty, loading, error).
7. Documentada con JSDoc en funciones pГәblicas.
8. Si involucra UI crГӯtica, **validada con el MCP de Playwright**.
